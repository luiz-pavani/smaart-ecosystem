## Key Patterns / Gotchas Discovered

### 1. Supabase join arrays
When you do `.select('academia:academia_id(id, nome, sigla)')`, Supabase TypeScript returns `academia` as an **array**, even for single-row FK joins. Always normalize at fetch time:

```ts
academia: Array.isArray(item.academia) ? item.academia[0] || null : item.academia
```

And type the interface as:
```ts
academia?: { id: string; nome: string; sigla: string; } | null
```

### 2. Supabase untyped client
The project uses no `Database` generic on the Supabase client, but explicit `.select('field1, field2')` strings DO create inferred types at the call site — so if you select a field that doesn't exist, TypeScript WILL catch it.

### 3. proxy.ts replaces middleware.ts
Do **NOT** create a `middleware.ts` — Vercel build will fail with:
> "Both middleware.ts and proxy.ts detected. Use proxy.ts only."

### 4. Vercel non-fatal warning (repeats every build)
> `⚠ Both outputFileTracingRoot and turbopack.root are set, but they must have the same value.`

Non-blocking. Located in `next.config.js` — `outputFileTracingRoot` and `turbopack.root` values differ. Can fix later.
